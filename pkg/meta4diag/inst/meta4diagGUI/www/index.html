Correlation
Description There are 3 strategies to define a PC prior for the correlation parameter ρ. To see the details, please choose from the following strategy:
Strategy 1 is defined using
P(ρ < ρ0 | ρ0) = ω,
P(ρ < u1 | ρ0) = α1,
where ρ0 is the base model, i.e. one believes that ρ is around ρ0; u1 is a value that is smaller than ρ0; α1 is the probability of the left tail event.
Strategy 2 is defined using
P(ρ < ρ0 | ρ0) = ω,
P(ρ > u2 | ρ0) = α2,
where ρ0 is the base model, i.e. one believes that ρ is around ρ0; u2 is a value that is larger than ρ0; α2 is the probability of the right tail event.
Strategy 3 is defined using
P(ρ < u1 | ρ0) = α1,
P(ρ > u2 | ρ0) = α2,
where ρ0 is the base model, i.e. one believe that ρ is around ρ0; u1 is a value that is smaller than ρ0; α1 is the probability of the left tail event; u2 is a value that is larger than ρ0; α2 is the probability of the right tail event.

Description The number of samples that are generated from an approximated posterior of a fitted model. The samples are used to estimate the (log) positive likelihood ratio (L)LR+, the (log) negative likelihood ratio (L)LR-, the (log) diagnostic odds ratio (L)DOR and the area under the ROC curve (AUC). If the number changes, the model needs to be re-fitted.
